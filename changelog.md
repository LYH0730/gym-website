## 2026년 1월 29일

### ✨ 기능 개선: 트레이너별 스케줄 뷰 UI 개선

- **문제점**: 트레이너를 선택하여 스케줄을 볼 때, 상단에 단순한 텍스트만 표시되어 정보가 부족하고 시각적으로 매력적이지 않았습니다.
- **원인**: `Schedule.jsx`에서 선택된 트레이너의 뷰를 렌더링할 때, 간단한 `<p>` 태그를 사용하여 텍스트만 보여주고 있었습니다.
- **해결 과정 및 결과**:
  1.  `Schedule.jsx`를 수정하여, 트레이너가 선택되었을 때 표시되는 헤더 영역의 구조를 변경했습니다. 새로운 헤더는 트레이너의 이미지, 이름, 그리고 부제목을 포함하는 컨테이너로 교체되었습니다.
  2.  `Schedule.css`에 `.trainer-schedule-header`와 관련 자식 요소들에 대한 스타일을 추가했습니다. 이 스타일은 이미지를 원형으로 만들고, 테두리와 그림자 효과를 추가하며, 텍스트의 크기와 색상을 조절하여 시각적으로 풍부한 헤더를 구현합니다.
- **효과**: 이제 사용자가 스케줄 탭에서 특정 트레이너를 클릭하면, 해당 트레이너의 사진과 이름이 포함된 멋진 헤더가 표시되어 사용자 경험(UX)이 향상되었습니다.

### 🐞 버그 수정: 문의 페이지 연락처 정보 번역 오류 해결

- **문제점**: 문의 페이지의 연락처 정보(주소, 전화번호, 이메일)가 언어 설정이 영어일 때도 한글로 고정되어 표시되는 문제가 있었습니다.
- **원인**: 연락처 정보가 `Contact.jsx` 컴포넌트에 직접 텍스트로 하드코딩되어 있어, `i18next` 번역 시스템의 영향을 받지 않았습니다.
- **해결 과정 및 결과**:
  1.  `public/locales/en/translation.json`과 `public/locales/ko/translation.json` 파일에 각각 영어와 한국어 연락처 정보에 대한 번역 키(`phone`, `email`, `address`)를 추가했습니다.
  2.  `Contact.jsx`에서 하드코딩된 텍스트를 `i18next`의 `t()` 함수를 사용하는 코드로 교체하여, 언어 설정에 따라 동적으로 텍스트가 변경되도록 수정했습니다.
- **효과**: 이제 사용자가 언어 설정을 변경하면 문의 페이지의 연락처 정보가 해당 언어에 맞게 올바르게 표시됩니다.

### ✨ 신규 기능: 문의하기 폼 이메일 전송 기능 구현

- **문제점**: 기존 문의하기 폼은 사용자에게 "전송 완료" 알림만 보여줄 뿐, 실제 이메일을 보내는 기능이 없었습니다.
- **원인**: `Contact.jsx`의 `handleSubmit` 함수에 이메일 전송 로직이 구현되어 있지 않았습니다.
- **해결 과정 및 결과**:
  1.  `@emailjs/browser` 라이브러리를 설치하여 이메일 전송 기능을 추가했습니다.
  2.  `Contact.jsx` 파일에 사용자가 요청한 새로운 연락처 정보(전화번호, 이메일, 주소)를 업데이트했습니다.
  3.  폼 제출 시 EmailJS의 `sendForm` API를 호출하도록 `handleSubmit` 함수를 수정했습니다.
  4.  EmailJS 연동에 필요한 `SERVICE_ID`, `TEMPLATE_ID`, `PUBLIC_KEY`를 `.env` 파일에 환경 변수로 추가하여, 키 값들이 코드에 직접 노출되지 않도록 보안을 강화했습니다.
  5.  사용자에게 EmailJS 계정 설정 및 템플릿 생성 방법을 안내하여 최종 연동을 완료할 수 있도록 지원했습니다.
- **효과**: 이제 사용자가 문의하기 폼을 작성하고 제출하면, 지정된 이메일 주소로 실제 문의 내용이 전송되어 관리자가 확인할 수 있게 되었습니다.

### ✨ 기능 개선: 카카오맵 스크롤 확대/축소 비활성화

- **문제점**: 웹페이지 전체를 스크롤할 때 마우스 커서가 지도 위에 있으면 의도치 않게 지도가 확대/축소되어 사용자 경험을 저해했습니다.
- **원인**: 카카오맵 API의 기본 설정에서 마우스 휠을 이용한 확대/축소 기능(`scrollwheel`)이 활성화되어 있었기 때문입니다.
- **해결 과정 및 결과**: `gym-website/src/components/KakaoMap.jsx` 파일 내 지도 옵션(`mapOption`)에 `scrollwheel: false` 속성을 추가했습니다. 이를 통해 지도 위에서 마우스 휠을 움직여도 확대/축소 기능이 동작하지 않도록 수정했습니다.
- **효과**: 이제 사용자는 페이지 스크롤 시 지도 영역을 지나가더라도 스크롤 흐름이 끊기지 않아 더 자연스러운 웹사이트 탐색이 가능해졌습니다.

## 2026년 1월 15일

### ✨ 기능 개선: 스케줄 관리 UI/UX 개선

- **문제점**: 기존에는 일반 사용자가 보는 스케줄 페이지와 관리자가 수정하는 페이지가 분리되어 있어, 관리자가 스케줄을 수정하기 위해 별도의 페이지로 이동해야 하는 번거로움이 있었습니다.
- **원인**: 사용자용 뷰와 관리자용 편집 기능이 별개의 컴포넌트(`Schedule.jsx`, `AdminSchedule.jsx`)로 구현되어 있었습니다.
- **해결 과정 및 결과**:
  1.  **`Schedule.jsx` 통합**: `AdminSchedule.jsx`에 있던 관리자 기능(인증 확인, 데이터 추가/수정/삭제 로직, 수정 폼 모달)을 `Schedule.jsx`로 모두 통합했습니다.
  2.  **조건부 렌더링**: 관리자로 로그인한 경우에만 '새 항목 추가', '수정', '삭제' 버튼이 스케줄 표에 나타나도록 `session` 상태를 확인하여 UI를 동적으로 변경했습니다.
  3.  **UI 스타일 개선**: `Schedule.css`에 관리자용 버튼과 컨트롤에 대한 스타일을 추가하여 기존 시간표 디자인과 자연스럽게 어우러지도록 했습니다.
  4.  **불필요한 파일 및 라우트 제거**: 기능이 통합됨에 따라 더 이상 필요 없어진 `AdminSchedule.jsx`, `AdminSchedule.css` 파일을 삭제하고, `App.jsx`와 `Navbar.jsx`에서 관련 라우트와 링크를 제거하여 코드를 정리했습니다.
- **효과**: 이제 관리자는 별도 페이지 이동 없이 스케줄을 보는 동시에 바로 수정할 수 있게 되어 작업 흐름이 크게 개선되고 사용자 경험(UX)이 향상되었습니다.
